# Design (Claude-owned)

Design is a named, sequenced step in the roadmap: **direction in Phase 0, surface mockups gate Phase 1.**

For a directory, design is mostly information architecture and data display, not marketing visuals. The filters cannot be designed until the taxonomy axes are locked, and the compatibility matrix cannot be designed until the schema exists. So design follows the schema and precedes the build.

## Contents

- [brand/brand-direction.md](brand/brand-direction.md) — brand identity, positioning, palette, typography, voice
- [brand/tokens/](brand/tokens/) — design tokens (color, type scale, radii, spacing) as the single source for the component system
- [mockups/](mockups/) — Phase 0.5 surface mockups (Pencil), reviewed before any Phase 1 build

## The one rule that shapes everything

The directory gets its **own** visual identity. The single deliberate family resemblance to en:twine is a **red/white palette**. Everything else (name, logo, typography, component look, layout) is fresh. The link to en:twine is endorsement and attribution only ("Works by en:twine"), never visual inheritance. A neutral, trustworthy directory must not look like an en:twine marketing site; its own identity is what sells the neutral-first promise.
