# Surface mockups (Phase 0.5)

This is the **design gate** before any Phase 1 build. Claude mocks the key surfaces in Pencil; they are reviewed before Codex scaffolds real UI.

Surfaces to mock (from ENT-412 §9a):

1. **Catalogue / filter** — the browse surface with the multi-axis filters (JTBD, integration surface, effort, model, hosting, tags).
2. **Tool card** — the repeating unit in the catalogue.
3. **Compatibility matrix** — tool x platform, method, effort. The moat, made visible.
4. **Match-and-scope flow + effort readout** — the headline lead magnet: a self-serve tool that returns a work-needed estimate.
5. **Tool detail** — the full record for one tool, including verification state and "Works by en:twine" endorsement.

Blocked on: the taxonomy axes and content-collection schema being locked (design follows schema). Exports land here as static images plus the Pencil source reference.
